# 油价调整日期配置说明

## 概述

本系统使用基于中国节假日的工作日计算来生成油价调整日期。每年元旦后，国务院会发布当年的节假日安排，需要更新配置文件。

## 文件说明

- `holidays.js` - 节假日配置和工作日计算工具

## 每年更新步骤

### 1. 获取官方节假日安排

每年12月或1月初，国务院会发布下一年的节假日安排：
- 官方来源：国务院办公厅
- 包含信息：法定节假日、调休安排

### 2. 更新 `holidays.js` 中的 `CHINA_HOLIDAYS`

在 `CHINA_HOLIDAYS` 对象中添加新年份的数据：

```javascript
export const CHINA_HOLIDAYS = {
  2025: [
    { name: '元旦', dates: ['2025-01-01'] },
    { name: '春节', dates: ['2025-01-28', '2025-01-29', ...] },
    // ... 其他节假日
  ],
  2026: [
    // ... 2026年数据
  ],
  2027: [
    // 新增2027年数据
    { name: '元旦', dates: ['2027-01-01', '2027-01-02', '2027-01-03'] },
    { name: '春节', dates: ['2027-02-xx', ...] },
    // ... 根据官方公告填写
  ],
};
```

### 3. 验证油价调整日期

系统会自动根据节假日计算调价日期：
- 规则：每10个工作日调整一次
- 跳过：周末和法定节假日
- 起始日期：每年第一个调价窗口（通常在1月初）

验证方法：
1. 构建项目：`pnpm run build`
2. 访问油价日历页面
3. 检查标记的调价日期是否与官方公告一致

### 4. 对比官方公告

油价调整时间通常由国家发改委在年初公布，可以通过以下途径获取：
- 国家发改委官网
- 各省发改委官网（如：广东省发改委）
- 新闻媒体报道

对比要点：
- 检查前3-4个调价日期是否与官方公告一致
- 如有偏差，需要检查节假日配置是否准确

## 调价日期计算逻辑

### 核心函数

1. **`isWorkday(date)`** - 判断是否为工作日
   - 排除周末（周六、周日）
   - 排除法定节假日

2. **`addWorkdays(startDate, workdaysToAdd)`** - 从指定日期增加N个工作日
   - 自动跳过周末和节假日
   - 返回第N个工作日的日期

3. **`generateAdjustmentDates(startYear, startDate, count)`** - 生成调价日期列表
   - 从起始日期开始
   - 每隔10个工作日生成一个调价日期
   - 默认生成25个（覆盖全年）

### 工作原理示例

假设2027年1月6日是第一个调价日：

```
2027-01-06 (周二) -> 调价日 #1
   ↓ +10个工作日
2027-01-20 (周二) -> 调价日 #2
   ↓ +10个工作日（跳过春节假期）
2027-02-xx (周x) -> 调价日 #3
```

## 特殊情况处理

### 1. 春节长假

春节通常有7-8天假期，会导致调价窗口延后较多天。系统会自动跳过这些假期。

### 2. 国庆长假

国庆通常有7-8天假期，处理方式同春节。

### 3. 调休

部分节假日会涉及调休（周末上班），但油价计算通常不考虑调休，仍按自然日的周末跳过。
如官方公告与系统计算有偏差，以官方公告为准，需要手动调整。

## 常见问题

### Q: 为什么不直接写死所有调价日期？

A: 因为节假日每年不同，自动计算比手动维护更可靠。只需要更新节假日配置，系统会自动生成所有调价日期。

### Q: 如果计算的日期和官方公告不一致怎么办？

A: 以官方公告为准。可能的原因：
1. 节假日配置有误
2. 官方有特殊调整
3. 起始日期选择有误

解决方案：检查并修正节假日配置，或在 `KNOWN_ADJUSTMENT_DATES` 中手动覆盖该年份的数据。

### Q: 跨年度的调价日期如何处理？

A: 系统会自动处理跨年。`getNextAdjustmentFromToday()` 函数会查找下一年的调价日期。

## 技术细节

### 节假日数据结构

```javascript
{
  name: '春节',  // 节假日名称
  dates: [       // 假期日期数组（字符串格式：YYYY-MM-DD）
    '2027-02-12',
    '2027-02-13',
    // ...
  ]
}
```

### 日期格式

所有日期使用 ISO 8601 格式：`YYYY-MM-DD`

### 性能优化

- 使用 Set 数据结构快速查找节假日
- 一次性生成全年调价日期，避免重复计算
