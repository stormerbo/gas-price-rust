# 节假日管理系统使用指南

## 功能概述

本系统实现了一个完整的节假日和油价调整日期管理方案，解决了硬编码维护困难的问题。

### 核心特性

✅ **自动同步节假日**：从 GitHub 爬取官方节假日数据
✅ **工作日计算**：基于节假日自动计算工作日
✅ **自动生成调价日期**：根据节假日和调价规则自动生成全年调价日期
✅ **可配置规则**：支持修改调价间隔（每N个工作日）
✅ **可分发应用**：用户可自行同步和维护数据

## 使用步骤

### 1. 首次使用：同步节假日数据

1. 启动应用后，点击导航栏的 **"⚙️ 系统设置"**
2. 在 **"节假日管理"** 标签页，点击 **"同步节假日数据"** 按钮
3. 系统会从 GitHub (NateScarlet/holiday-cn) 自动下载前一年、当年、下一年的节假日数据
4. 同步完成后，会显示成功消息，例如："成功同步3个年份（2025, 2026, 2027），共450条记录"

### 2. 查看节假日

同步完成后，页面会自动显示当年的节假日列表：
- **日期**：节假日日期（YYYY-MM-DD）
- **节假日名称**：如"春节"、"国庆节"等
- **是否休息**：显示"休息"（绿色）或"上班"（红色）

### 3. 配置调价规则

切换到 **"调价规则配置"** 标签页：

1. **工作日间隔**：设置每隔多少个工作日调整一次（默认10个工作日）
2. **首次调价日期**：设置每年的第一个调价日期
   - 2025年默认：2025-01-02
   - 2026年默认：2026-01-06
3. 点击 **"保存配置"** 保存更改

### 4. 查看调价日期

保存配置后，页面下方会显示当年的所有调价日期：
- **序号**：第几次调价（第1次、第2次...）
- **调价日期**：自动计算的调价日期

系统会自动跳过周末和节假日，确保调价日期都是工作日。

### 5. 在日历中查看

点击导航栏的 **"📅 油价日历"**：
- 调价日期会在右上角显示 **橙色圆点** 标记
- 已发生的调价显示 **"↑涨"** 或 **"↓跌"**（红色/绿色）
- 未来的调价日期显示 **"待调价"**（橙色）

## 数据源

### 节假日数据

- **来源**：https://github.com/NateScarlet/holiday-cn
- **维护**：自动每日更新
- **格式**：JSON
- **覆盖年份**：2007年至今

### 数据存储

所有数据存储在本地 SQLite 数据库：
- 位置：`~/.gas_price/data/gas_prices.db`
- 表：
  - `holidays` - 节假日数据
  - `adjustment_settings` - 调价规则配置

## 年度维护

### 每年需要做什么？

每年元旦后（约1月初）：

1. **同步节假日**：
   - 进入"系统设置" → "节假日管理"
   - 点击"同步节假日数据"
   - 系统会自动获取新一年的节假日

2. **验证调价日期**（可选）：
   - 切换到"调价规则配置"标签
   - 查看生成的调价日期列表
   - 对比国家发改委公布的官方调价时间表
   - 如有偏差，检查"首次调价日期"配置是否正确

3. **更新首次调价日期**（如需要）：
   - 如果新一年的首次调价日期还未配置
   - 在"调价规则配置"中添加新年份的首次调价日期
   - 例如：2027年首次调价日期

### 无需维护的部分

- ✅ 节假日数据：自动从 GitHub 同步
- ✅ 调价日期计算：基于节假日自动生成
- ✅ 工作日判断：算法自动处理
- ✅ 跨年处理：系统自动识别

## API 文档

### 节假日相关

```
GET  /api/v1/holidays?year=2026          # 获取指定年份节假日
POST /api/v1/holidays/sync                # 同步节假日数据
GET  /api/v1/holidays/adjustment-dates?year=2026  # 获取调价日期
```

## 常见问题

### Q: 同步失败怎么办？

A: 可能原因：
1. **网络问题**：检查网络连接，确保能访问 GitHub
2. **数据源问题**：GitHub 可能暂时不可用，稍后重试
3. **数据格式变更**：数据源格式可能有更新，需要更新应用

解决方法：
- 重试几次
- 检查网络代理设置
- 查看应用日志获取详细错误信息

### Q: 调价日期不准确怎么办？

A: 检查步骤：
1. 确认节假日数据已同步
2. 检查"首次调价日期"配置是否正确
3. 查看国家发改委公布的官方调价时间表
4. 如有偏差，调整"首次调价日期"并重新保存

### Q: 如何添加新年份的配置？

A: 当前系统支持 2025-2026 年。如需添加 2027 年：
1. 同步节假日数据（会自动下载2027年数据）
2. 后端代码需要修改以支持新年份（未来会优化为动态支持）

### Q: 数据库文件在哪里？

A: 
- macOS/Linux: `~/.gas_price/data/gas_prices.db`
- Windows: `C:\Users\<用户名>\.gas_price\data\gas_prices.db`

### Q: 如何重置数据？

A: 删除数据库文件即可：
```bash
# macOS/Linux
rm ~/.gas_price/data/gas_prices.db

# Windows
Remove-Item $env:USERPROFILE\.gas_price\data\gas_prices.db
```
应用会在下次启动时自动创建新数据库。

## 技术细节

### 工作日计算算法

```
1. 从起始日期开始
2. 每次前进1天
3. 检查是否为周末（周六/周日）
4. 检查是否为节假日
5. 如果是工作日，计数器+1
6. 重复直到计数器达到目标天数
```

### 调价日期生成

```
1. 读取节假日数据
2. 从"首次调价日期"开始
3. 每隔N个工作日生成一个调价日期
4. 持续生成直到覆盖全年（通常25次）
```

### 数据同步流程

```
1. 构建HTTP请求到GitHub
2. 下载 {year}.json 文件
3. 解析JSON数据
4. 存储到 holidays 表
5. 使用 INSERT OR REPLACE 避免重复
```

## 优势对比

### 旧方案（硬编码）
❌ 每年需要手动更新代码
❌ 需要重新编译和发布应用
❌ 用户无法自行维护
❌ 容易出错（手动计算工作日）

### 新方案（数据库+爬虫）
✅ 用户一键同步，无需更新代码
✅ 无需重新发布应用
✅ 数据存储在本地数据库
✅ 算法自动计算，准确可靠
✅ 支持配置调价规则
✅ 可分发应用，用户自主维护
