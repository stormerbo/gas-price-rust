# 闪退问题测试结果总结

## 测试环境
- 系统：macOS
- 应用：中国汽油价格管理 v0.1.0

## 测试结果

### ✅ 测试 1: 正常启动
**结果**: 成功
- 应用可以正常启动
- 后端服务器绑定到 8080 端口
- 前端界面正常显示

### ❌ 测试 2: 隔离属性（模拟新用户下载）
**结果**: 失败 - 成功复现闪退
- 错误信息：`_LSOpenURLsWithCompletionHandler() failed with error -128`
- 这是 macOS Gatekeeper 阻止未签名应用
- **这很可能是用户遇到的问题**

### ✅ 测试 3: 端口冲突
**结果**: 待测试
- 需要占用 8080-8090 所有端口

### ✅ 测试 4: 缺少资源文件
**结果**: 不会导致闪退
- 即使删除 static 目录，应用仍能启动
- 只是界面无法加载

## 结论

**最可能的原因**: macOS Gatekeeper 安全机制

用户下载应用后，macOS 会自动添加隔离属性（quarantine），导致：
1. 首次打开时提示"应用已损坏"
2. 即使用户绕过提示，应用也可能因权限问题闪退

## 解决方案

### 方案 1: 用户手动移除隔离属性（当前方案）
```bash
xattr -cr /Applications/中国汽油价格管理.app
```

优点：
- 免费
- 简单

缺点：
- 用户体验差
- 需要使用终端

### 方案 2: 代码签名（推荐）
需要：
- Apple Developer 账号（$99/年）
- Developer ID Application 证书
- 公证（notarization）

优点：
- 用户可以直接打开
- 专业
- 通过 Gatekeeper 验证

缺点：
- 需要付费
- 配置复杂

### 方案 3: 提供更好的安装脚本（当前已实现）
- 一键安装脚本自动移除隔离属性
- 提供详细的安装说明
- 包含诊断工具

## 建议

1. **短期**: 使用当前的安装包方案
   - 包含一键安装脚本
   - 提供详细说明文档
   - 包含诊断工具

2. **长期**: 考虑购买 Apple Developer 账号进行代码签名
   - 提升用户体验
   - 增加应用可信度
   - 避免 Gatekeeper 问题
