# 构建说明

## 问题诊断

### 闪退原因
如果应用在你的电脑上正常运行，但在其他人的电脑上闪退，最可能的原因是：

**架构不匹配**
- 你的 Mac: Apple Silicon (M1/M2/M3) - arm64
- 其他人的 Mac: Intel - x86_64

单架构应用无法在不同架构的 Mac 上运行。

## 解决方案：构建通用二进制

### 方法 1: 使用快速脚本（推荐）

```bash
# 构建通用版本并创建安装包（一步完成）
./build_universal.sh
./create_universal_installer.sh
```

生成文件：
- `中国汽油价格管理_通用安装包_v0.1.0.dmg` (约 25MB)

### 方法 2: 使用更新后的 build_package.sh

```bash
# 现在默认构建通用版本
./build_package.sh
```

## 验证通用二进制

```bash
# 检查应用支持的架构
lipo -info target/universal/中国汽油价格管理.app/Contents/MacOS/中国汽油价格管理

# 应该显示：
# Architectures in the fat file: ... are: x86_64 arm64
```

## 文件大小对比

| 版本 | 大小 | 支持架构 |
|------|------|----------|
| 单架构 (arm64) | ~15MB | 仅 Apple Silicon |
| 单架构 (x86_64) | ~20MB | 仅 Intel |
| 通用版本 | ~25MB | Intel + Apple Silicon |

## 分发建议

### 推荐方案
使用通用版本安装包：
- ✅ 一个文件支持所有 Mac
- ✅ 用户无需关心架构
- ✅ 避免闪退问题

### 备选方案
如果文件大小是问题，可以分别提供：
- Intel 版本 (x86_64)
- Apple Silicon 版本 (arm64)

但需要用户自己选择正确的版本。

## 构建脚本说明

### build_universal.sh
- 构建 Intel 和 Apple Silicon 两个版本
- 使用 `lipo` 合并为通用二进制
- 创建独立的 DMG

### create_universal_installer.sh
- 基于通用应用创建安装包
- 包含一键安装脚本
- 包含诊断工具和说明文档

### build_package.sh（已更新）
- 现在默认构建通用版本
- 自动安装所需的构建目标
- 验证最终二进制的架构

## 常见问题

### Q: 为什么通用版本更大？
A: 包含了两个架构的代码，大小约为单架构的 1.5-2 倍。

### Q: 通用版本性能会差吗？
A: 不会。macOS 会自动选择匹配的架构运行，性能与单架构版本相同。

### Q: 可以只构建 Intel 版本吗？
A: 可以，使用：
```bash
cargo tauri build --target x86_64-apple-darwin
```

### Q: 如何知道用户的 Mac 是什么架构？
A: 让用户运行：
```bash
uname -m
# arm64 = Apple Silicon
# x86_64 = Intel
```

## 推荐工作流

1. **开发阶段**: 使用 `cargo tauri dev` 快速测试
2. **测试阶段**: 使用 `cargo tauri build` 构建当前架构
3. **发布阶段**: 使用 `./build_universal.sh` 构建通用版本
4. **分发阶段**: 使用 `./create_universal_installer.sh` 创建安装包
